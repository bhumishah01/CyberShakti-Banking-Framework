## Extra: Real-World Scenario Walkthrough
A rural user creates a transaction in weak-network conditions. The system stores it locally, evaluates it for fraud, gives an immediate decision, and later synchronizes the action to the central server. The bank/admin portal then reflects the final monitored state.
