# Challenges

## Problems Faced During Development
- presenting offline-first states clearly in the UI
- keeping many pages visually and functionally consistent
- preserving explainability as fraud and analytics features grew
- moving from localhost to a stable public deployment
- converting documentation into a proper final report

## Limitations of the System
- biometric verification is prototype-level
- no real banking rail integration
- no production notification infrastructure
- fraud scoring is lightweight and explainable rather than ML-heavy

## Extra: Failure Handling Summary
- no internet -> local save
- risky activity -> hold or block
- sync failure -> queue and retry
- suspicious user -> admin alert and control
